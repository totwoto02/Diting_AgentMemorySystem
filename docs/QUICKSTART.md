# MFS 快速开始（10 分钟入门）

## 什么是 MFS？

MFS (Memory File System) 是 **AI 记忆的 Git + NTFS**，提供：

- ✅ **文件系统概念**：用路径管理记忆（如 `/person/用户朋友/preferences`）
- ✅ **自动切片**：长文本自动切分，无需担心长度限制
- ✅ **自动还原**：读取时自动拼装，用户无感知
- ✅ **智能搜索**：知识图谱扩展，找到相关记忆
- ✅ **防幻觉**：WAL 日志 + 证据校验（Phase 2 后续）

---

## 1. 安装（1 分钟）

```bash
pip install mfs-memory
```

验证安装：
```bash
mfs-check-install
```

看到 `🎉 所有检查通过！` 即安装成功。

---

## 2. 基本使用（5 分钟）

### 写入记忆

**短文本**（<2000 字）：
```
使用 mfs_write 工具：
path: /test/notes
type: NOTE
content: 这是短笔记
```

**长文本**（>2000 字）：
```
使用 mfs_write 工具：
path: /test/long_doc
type: DOCUMENT
content: [5000 字的长文档...]

返回：已创建：/test/long_doc (自动切片=3 片)
```

### 读取记忆

```
使用 mfs_read 工具：
path: /test/long_doc

返回：完整原文（自动还原）
```

### 搜索记忆

```
使用 mfs_search 工具：
query: 用户朋友

返回：
找到 2 条结果:
- /person/用户朋友 (CONTACT): 用户朋友的个人资料...
- /person/用户朋友/preferences (NOTE): 喜欢游戏...

💡 提示：搜索"用户朋友"时，可能也关心：游戏、游戏角色
```

## 3. 进阶功能（4 分钟）

### 知识图谱

写入记忆时自动构建概念关联：

```
写入 1: "用户朋友喜欢游戏"
写入 2: "游戏角色是游戏的角色"
写入 3: "忠犬类型的角色很受欢迎"

自动构建关联：
用户朋友 → 游戏 → 游戏角色 → 忠犬
```

搜索时自动扩展：
```
搜索"用户朋友" → 提示"可能也关心：游戏、游戏角色"
```

### 路径规范

推荐的路径结构：
```
/person/{name}          # 人物资料
/person/{name}/prefs    # 人物偏好
/work/{project}         # 项目记录
/work/{project}/rules   # 项目规则
/notes/{category}       # 笔记分类
```

---

## 4. 最佳实践

### ✅ 推荐

- 使用有意义的路径（如 `/person/用户朋友/preferences`）
- 选择正确的类型（NOTE/RULE/CODE/TASK/CONTACT/EVENT）
- 长文本直接写入（自动切片）
- 定期搜索验证记忆存在

### ❌ 避免

- 路径不使用 `/` 开头
- 类型拼写错误
- 频繁删除记忆（使用 archived 状态代替）

## 常见问题

**Q: 切片会影响性能吗？**  
A: 不会。切片在写入时进行，读取时自动还原，用户无感知。

**Q: 知识图谱需要手动维护吗？**  
A: 不需要。自动从记忆内容提取关键词并构建关联。

**Q: 如何备份记忆？**  
A: MFS 使用 SQLite 存储，直接备份数据库文件即可。
